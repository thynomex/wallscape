# Security Policy

## Supported versions

Security fixes are provided for the latest released version and the current `main` branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting feature on the Security tab of this repository. Include the affected version, impact, reproduction steps, and any suggested mitigation.

Avoid including credentials, private wallpaper files, or unrelated personal information. Maintainers will acknowledge a complete report as soon as practical and coordinate disclosure after a fix is available.
